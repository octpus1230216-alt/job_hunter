# 边界规则 — 半自动找工作工具

## 1. 核心边界

### 1.1 半自动边界
本工具的核心边界是"发现自动化，投递人工化"。

**✅ 工具会做的：**
- 从公开平台搜索职位信息
- AI 分析简历与岗位的匹配度
- 根据 JD 定制简历和求职信
- 生成直达搜索链接供用户跳转到招聘平台
- 通过 Chrome 扩展在用户真实浏览器中自动翻页采集（用户主动浏览）
- 通过 Tampermonkey 脚本收集用户浏览时看到的 JD（备选方案）
- 所有采集行为发生在用户自己的浏览器中，请求由用户 IP 发出
- 每日自动搜索并生成精选推荐

**❌ 工具不会做的：**
- 自动在招聘平台上注册/登录账号
- 自动填写申请表并提交
- 自动发送消息给招聘方
- 模拟用户在平台上进行任何交互操作
- 绕过平台的反爬机制或验证码

### 1.2 数据边界
- 所有用户数据（简历、API Key、配置、搜索结果）仅存储在本地磁盘
- 简历解析时，文本内容发送到用户选择的 AI API（DeepSeek/OpenAI/Ollama）
- 使用 Ollama 本地模型时，所有数据完全不出用户电脑
- 浏览器采集脚本通过 localhost 通信，数据不经过任何外部服务器
- 不要求用户提供任何第三方平台（Boss直聘/猎聘等）的登录凭证

### 1.3 平台使用边界
在抓取招聘平台数据时，遵循以下规则：
- **只读访问**：仅获取公开可访问的职位信息，不修改、不提交
- **频率控制**：JobSpy 搜索内置 sleep 间隔（1 秒），避免触发限流
- **平台偏好**：优先使用 Indeed（反爬限制最宽松），Glassdoor 已知不稳定
- **LinkedIn 警告**：需要代理才能稳定使用，单 IP 约第 10 页触发限流
- **国内平台**：不进行自动爬取。通过浏览器脚本（用户自己浏览器发请求）收集数据

## 2. 功能边界

### 2.1 搜索边界

| 渠道 | 覆盖范围 | 限制 |
|------|---------|------|
| 🌍 海外 Indeed | 74 个国家，反爬宽松 | 中文关键词效果差，部分国家可能无结果 |
| 🌍 海外 LinkedIn | 全球 | 需要代理，中国搜索结果可能混入美国岗位 |
| 🌍 海外 Glassdoor | 部分国家 | API 不稳定，中国不可用，不建议 |
| 🏠 国内 Boss直聘 | 中国大陆 | 仅生成直达链接，不自动抓取 |
| 🏠 国内 猎聘 | 中国大陆 | 仅生成直达链接，不自动抓取 |
| 🤖 AI 全网搜索 | DuckDuckGo | 结果质量参差，依赖 AI 二次筛选 |
| 📋 手动粘贴 | 任意平台 | 依赖用户手动复制 JD |

### 2.2 匹配边界
- **匹配不是绝对判断**：AI 评分基于语义分析，仅供参考。高分不等于一定合适，低分不等于不合适
- **跨行业匹配**：能力可迁移分析基于 LLM 的"语义桥接"，可能漏掉不常见的跨行业路径
- **不编造信息**：简历定制只调整措辞和排序，不允许 AI 编造不存在的经历或技能
- **技能表如实反映**：JD 要求但简历中没有的技能，会在"技能"部分标注但不夸大

### 2.3 语言边界
- **双语生成**：简历和求职信可同时生成中英文版本
- **搜索关键词**：AI 同时生成中英文关键词，海外平台用英文，国内平台用中文
- **海外搜索中文问题**：中文简历的关键词直接用于英文平台搜索效果差，需 AI 生成英文关键词
- **混合场景**：国内公司海外分公司可能需要中英混合简历，工具会同时生成两个版本

### 2.4 薪资边界
- **不降薪策略**：AI 职业定位和岗位匹配时，过滤薪资低于当前水平的岗位
- **薪资估计**：基于简历信息和市场数据的估计，非精确值
- **跨行业薪资**：可迁移方向的薪资可能与当前行业有差异，工具会标注

### 2.5 公司发现边界
- **不保证发现所有公司**：公司发现依赖 AI 知识库和搜索结果，可能遗漏新公司或冷门公司
- **公司信息不完整**：公司规模、文化等信息来自公开数据，可能过时或不准确
- **不提供公司内部信息**：无法获取未公开的招聘计划或内部推荐信息

## 3. 技术边界

### 3.1 依赖边界
| 依赖 | 必要性 | 替代方案 |
|------|--------|---------|
| Python 3.13+ | 必需 | 无 |
| streamlit | 必需 | 无 |
| python-jobspy | 海外搜索必需 | 可只用 AI 全网搜索 |
| LLM API (DeepSeek/OpenAI/Ollama) | 核心功能必需 | Ollama 本地免费 |
| Tampermonkey | 浏览器采集必需 | 可只用手动粘贴 |
| Flask | 浏览器采集接收器必需 | 同上 |

### 3.2 性能边界
- **单次搜索上限**：海外平台单次约 50 个结果，避免触发限流
- **匹配批处理上限**：单次最多匹配 30 个岗位（LLM API 调用次数限制）
- **AI 全网搜索上限**：每个关键词最多 15 个结果，每次最多 12 个关键词
- **后台任务超时**：搜索任务无固定超时，但用户可随时取消

### 3.3 存储边界
- **简历解析缓存**：基于文件 SHA256 哈希，简历更新后自动重新解析
- **搜索结果**：不自动清理，用户可手动管理
- **每日精选**：按日期存储，建议定期清理超过 30 天的记录
- **后台任务文件**：完成后保留，用户导入后清理

### 3.4 安全边界
- **API Key 存储**：明文存储在 `config.yaml` 中（本地文件），建议使用环境变量
- **数据传输**：LLM API 调用通过 HTTPS 加密
- **本地通信**：浏览器采集器通过 HTTP (localhost)，不加密（仅本地）
- **无远程访问**：Streamlit 默认绑定 0.0.0.0:8501，同一局域网内可访问

## 4. 使用边界

### 4.1 运行环境
- **操作系统**：Windows（主要测试环境）、macOS/Linux（理论兼容）
- **Streamlit 服务**：需要在终端中持续运行，关闭终端则服务停止
- **后台搜索**：需要单独终端运行 `python search_worker.py`
- **每日精选**：需要配置 Windows 任务计划程序或手动运行

### 4.2 网络要求
- **LLM API**：需要互联网连接（使用 Ollama 本地模型除外）
- **海外搜索**：需要互联网连接，部分平台可能需要代理
- **AI 全网搜索**：需要互联网连接
- **浏览器采集**：仅 localhost，不需要互联网

### 4.3 数据持久化
- **Web 应用刷新**：Streamlit session_state 会重置，但磁盘数据不丢失
- **简历自动加载**：`resume_parsed.json` 存在时自动加载
- **审核列表**：当前存在 session_state，刷新会丢失（待改进为持久化）
- **生成结果**：保存为文件，刷新不丢失
- **投递记录**：保存为 JSON 文件，刷新不丢失

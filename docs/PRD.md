# 产品需求文档 (PRD) — 半自动找工作工具

## 1. 产品概述

### 1.1 产品定位
半自动求职辅助工具。帮助求职者高效完成找工作流程中最耗时的环节——职位发现、简历匹配、简历定制，但最终投递动作由用户手动完成。

### 1.2 核心价值
- **发现效率提升**：AI 自动从多个渠道发现职位，比手动浏览节省 80% 时间
- **匹配精准度**：LLM 语义匹配替代关键词匹配，避免漏掉"能力匹配但表述不同"的岗位
- **简历定制质量**：根据目标公司风格自动调整简历内容和语言，而非简单替换关键词
- **跨行业求职支持**：能力可迁移分析，帮助用户发现当前行业之外的就业机会

### 1.3 目标用户
- 有主动求职需求的职场人
- 考虑跨行业跳槽的求职者
- 需要同时投递国内和海外岗位的双语求职者
- 关注气候/ESG/国际发展等垂直领域的专业人士（当前用例）

### 1.4 产品原则
- **半自动而非全自动**：只帮用户发现和准备，不代替用户投递
- **数据本地化**：所有数据存储在用户本地，不上传任何第三方服务器
- **AI 辅助决策**：AI 提供建议和评分，最终决策权在用户
- **双语优先**：中英文简历和求职信同步生成

## 3. 非功能需求

### 3.1 性能
- API 调用超时时长：300 秒（已在 `.streamlit/config.toml` 配置）
- 搜索结果缓存：搜索结果保存为 JSON，避免重复请求
- 简历解析缓存：基于文件哈希缓存解析结果

### 3.2 安全性
- 所有用户数据（简历、配置、搜索结果）存储在本地磁盘
- API Key 通过密码输入框输入（`type="password"`）
- 浏览器采集脚本仅通过 localhost 通信（127.0.0.1:8765）
- 不执行自动投递，不模拟用户登录招聘平台

### 3.3 兼容性
- 操作系统：Windows（主要）、macOS/Linux（兼容）
- 浏览器：Chrome/Edge（Tampermonkey 脚本）
- Python：3.13+

### 3.4 可维护性
- Git 版本控制，每次修改提交记录
- CHANGELOG.md 记录版本变更
- 模块化架构，各模块职责清晰
